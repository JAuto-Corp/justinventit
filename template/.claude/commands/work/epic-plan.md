# /work:epic-plan

Read the `work` skill and execute the `/work:epic-plan` workflow.

**Steps**:
1. Read the epic issue and gather scope; identify the sprint and phase breakdown
2. Draft the sprint sequence and per-phase acceptance criteria (SPEC / SCENARIOS shape)
3. Record the plan in the state chain and point `docs/CURRENT_WORK.md` at the first sprint/phase
